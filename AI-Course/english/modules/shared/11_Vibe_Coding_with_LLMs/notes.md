# Vibe Coding with LLMs

Vibe coding is an exploratory, less-structured way of coding with an AI assistant.

## What it is
- Minimal, loose prompts.
- Focus on flow and creativity.
- Iterative exploration rather than strict constraints.

## When to use it
- Brainstorming and ideation.
- Creative coding and UI sketches.
- Prototypes and hackathons.

## Risks
- Less reproducible and harder to review.
- Lower determinism, higher variance.
- Weak guardrails unless you add them later.

## Best practice
Start loose to explore, then refactor into a structured, testable pattern. Keep both the exploratory prompt and the hardened prompt in your repo.
