This directory contains detailed examples and documentation for common development policies.

## Template Structure for Policy Examples

Each policy example file should follow this structure:

### Policy Rule Documentation Template
```markdown
## POLICY-ID - Policy Title Examples

### General Principles
- High-level explanation of the policy
- Why this policy matters
- What it aims to achieve

### Implementation Patterns
Generic patterns that work across languages:
- Pattern 1: Description and example
- Pattern 2: Description and example

### Best Practices
- Specific recommendations
- Common pitfalls to avoid
- Guidelines for implementation

### Language-Agnostic Examples
Examples using pseudocode or generic formats that apply to multiple languages.
```

### Documentation Guidelines

- Use language-agnostic examples when possible
- Focus on concepts rather than syntax
- Provide rationale for each recommendation
- Include both good and bad examples
- Keep explanations clear and concise
