### Template Instructions

**When to create this file:**
- Complex rules that need detailed examples
- Rules with multiple implementation approaches  
- Rules that vary significantly across languages
- Rules where examples significantly aid understanding

**When NOT to create this file:**
- Simple rules where README guidance suffices
- Rules with obvious implementation
- Language-agnostic rules with straightforward application

**Content Guidelines:**
1. Focus on practical, actionable examples
2. Include both good and bad examples for contrast
3. Explain the "why" behind each example
4. Connect examples back to the policy rule
5. Consider multiple scenarios and edge cases
6. Keep language-specific sections separate
7. Update examples when policies change

**File Naming Convention:**
- Use format: [PREFIX]-[NUMBER]-[rule-title-lowercase].md
- Example: JAVA-SEC-001-input-validation.md
- Example: BASE-TEST-001-test-coverage.md
